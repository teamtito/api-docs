RSVP

This allows you to react to an RSVP.

Respond to invitation

This will mark the status of the release invitation and if "yes" will create a registration according to what releases are associated with the release invitation.

curl --request POST \
  --url 'https://checkout.tito.io/rsvps/:release_invitation_slug/responses' \
  --header 'Accept: application/json' \
  --header 'Content-Type: application/json' \
  --data '{"status":"yes"}'

{
    "release_invitation": {
        "_type": "release_invitation",
        "id": 00001,
        "slug": "rsvp_abc",
        "first_name": "Alice",
        "last_name": "Smith",
        "email": "alice@example.com",
        "name": "Alice Smith",
        "status": "yes",
        "redeemed": true,
        "registration_id": 00002,
        "registration_slug": "reg_abc",
        "registration_reference": "ABC1",
        "releases": [
            {
                ...
            }
        ]
    }
}

POST https://checkout.tito.io/rsvps/:release_invitation_slug/responses

  • release_invitation_slug - The slug attribute of an Release Invitation.

Parameters

Parameter Type Description
status Required string Either "yes", "no" or "maybe"

Declining

Setting the status to "no" or "maybe" will not create a registration and will set the status acccordingly.
